=== SOTIF — ISO 21448:2022 ===

SOTIF = Safety of the Intended Functionality
SOTIF covers: performance limitations of correctly working systems (ADAS, cameras, radar)
SOTIF does NOT cover: hardware failures, random faults (that is ISO 26262)
SOTIF and ISO 26262 are DIFFERENT and COMPLEMENTARY standards.
SOTIF is NOT a replacement for ISO 26262.
SOTIF applies to: ADAS, automated driving, perception systems
SOTIF compliance requires: systematic scenario analysis, simulation, field testing.
SOTIF compliance CANNOT be claimed without validation evidence.

=== CAN Bus — ISO 11898 and OBD-II ===

OBD-II ADDRESSES (memorize these):
Broadcast request address: 0x7DF (send to all ECUs)
Engine ECU response: 0x7E8
Transmission ECU response: 0x7E9
Physical request to engine ECU: 0x7E0

WRONG ADDRESSES (hallucinations):
0x7FF = maximum CAN frame identifier, NOT an OBD-II address
0x100 = not a standard OBD-II address
0x200 = not a standard OBD-II address
0x7E = incomplete address (must be 0x7E0 through 0x7EF)

SECURITY RULE: Standard CAN bus has NO built-in authentication or encryption.
CAN bus is NOT inherently secure.
Security requires additional protocols: SecOC, CANsec, TLS over Ethernet.
Any claim that CAN has built-in encryption is a hallucination.

OBD-II PIDS (Mode 01):
0x0C = Engine RPM
0x0D = Vehicle Speed
0x05 = Coolant temperature
0x11 = Throttle position

=== AUTOSAR ===

AUTOSAR CLASSIC PLATFORM:
- Runs on microcontrollers
- Uses OSEK/VDX RTOS
- Communication: COM stack, CAN, LIN, FlexRay
- API: Classic COM API via RTE (Runtime Environment)
- Used for: engine control, ABS, airbags, body electronics
- Memory: static allocation only, no heap

AUTOSAR ADAPTIVE PLATFORM:
- Requires POSIX operating system (Linux or QNX)
- CANNOT run on bare-metal without OS
- Communication: ara::com (SOME/IP, DDS)
- Used for: ADAS, OTA updates, connected services, HPC ECUs
- Memory: dynamic allocation allowed (C++ STL)

CRITICAL RULES:
ara::com belongs ONLY to Adaptive Platform. Classic Platform does NOT use ara::com.
Saying "Classic AUTOSAR uses ara::com" is always a hallucination.
Saying "Adaptive Platform runs on bare-metal" is always a hallucination.
